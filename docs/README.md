# ドキュメントディレクトリ

このディレクトリには、プロジェクトの機能ごとの詳細なドキュメントが格納されています。

## ドキュメント一覧

- [image-feature.md](./image-feature.md) - 画像機能（アイキャッチ画像とマークダウン内画像）の解説
- [slash-commands.md](./slash-commands.md) - スラッシュコマンド機能（/imageなど）の解説

## ドキュメント作成ガイドライン

### 目的
コードの変更履歴や機能の実装詳細を記録し、将来のメンテナンスを容易にする。

### 作成タイミング
以下の場合に、必ずドキュメントを作成または更新してください：
- 新機能を追加した時
- 既存機能を大幅に変更した時
- バグ修正で仕様が変わった時
- データベーススキーマを変更した時

### ドキュメントのテンプレート

```markdown
# 機能名

## 概要
機能の目的と概要を簡潔に説明

---

## 実装の詳細

### 1. 機能A
**ファイル**: `path/to/file.ts:行番号`

\`\`\`typescript
// コードの例
\`\`\`

**機能**:
- 機能の説明

### 2. 機能B
（以下同様）

---

## データベーススキーマ（該当する場合）

### テーブル名
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | INTEGER | 主キー |

---

## 使用方法
実際の使い方の例

---

## 関連ファイル
- `path/to/file1.ts` - 説明
- `path/to/file2.tsx` - 説明

---

## 今後の改善案
1. 改善案1
2. 改善案2

---

**最終更新**: YYYY-MM-DD
```

### ファイル命名規則
- **形式**: `機能名.md`
- **命名**: ケバブケース（小文字 + ハイフン）
- **例**:
  - `image-feature.md` - 画像機能
  - `auth-system.md` - 認証システム
  - `markdown-rendering.md` - マークダウンレンダリング
  - `api-endpoints.md` - APIエンドポイント

### 記載内容の推奨事項
1. **コードの場所を明記**: `ファイル名:行番号` の形式で記載
2. **実際のコードを引用**: 重要な部分のコードを記載
3. **図解やテーブルを活用**: 理解しやすくする
4. **最終更新日を記載**: メンテナンス状況を把握できるように

### 更新ルール
- 機能を変更した際は、関連するドキュメントも必ず更新する
- 古い情報は削除せず、更新履歴として残す（必要に応じて）
- 最終更新日を必ず変更する
